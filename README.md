# -4
Анализ программного обеспечения. Построенных по архитектуре WEB-приложений.
1. Анализ рынка web-систем
На рынке чаще всего встречаются CRM, ERP, ECM/BPM, портальные системы, helpdesk, интернет-магазины и корпоративные личные кабинеты. Их объединяет то, что доступ идёт через браузер, а данные хранятся централизованно в СУБД, что упрощает сопровождение и удалённую работу.

С точки зрения архитектуры web-приложения делятся так:

Монолитные системы, где интерфейс, логика и доступ к данным находятся в одном решении; они проще в разработке, но хуже масштабируются.

Трёхзвенные клиент-серверные системы, где UI, бизнес-логика и БД разделены; это наиболее практичный вариант для корпоративных задач.

Микросервисные решения, где функции вынесены в независимые сервисы; они лучше подходят для крупных нагрузок и часто меняющихся процессов.

Serverless-решения, где часть логики выносится в облачные функции; они удобны для краткосрочных и событийных сценариев.

2. Возможности систем
Web-информационные системы обычно поддерживают авторизацию, разграничение прав, CRUD-операции, отчёты, поиск, интеграции через API и работу из любой точки сети. Для компании это означает уменьшение затрат на установку клиентского ПО, упрощение обновлений и централизованный контроль данных.

Для внедрения в компании можно выделить такие варианты:

Корпоративный портал для внутренних заявок и регламентов.

CRM для работы с клиентами и сделками.

Учет заявок и инцидентов в ИТ-службе.

Складской или кадровый web-кабинет.

Каталог документов или справочной информации.

Если нужен быстрый и надёжный запуск в небольшой или средней компании, чаще всего рационален вариант web-приложения в трёхзвенной архитектуре с IIS и SQL Server, потому что он хорошо сочетается с Windows-инфраструктурой и проще в администрировании.

3. Delphi 10.2, IIS и MS SQL Server
Для Delphi-проекта под IIS обычно используют ISAPI-модуль или серверный web-слой, который разворачивается как приложение IIS. В опубликованной инструкции по Delphi MVC Framework описан типовой сценарий: установка IIS, включение ISAPI Extensions/Filters, создание application pool, размещение DLL, настройка разрешений и запуск через URL.

Для работы с MS SQL Server в Delphi удобно использовать FireDAC; в примере Embarcadero показано подключение к MSSQL через временную connection definition и работу с запросами master-details, а также выполнение SQL-команд insert, update, delete и select. Это хорошо подходит для web-приложения, где серверная часть получает запросы, работает с БД и возвращает HTML или JSON.

4. Пример приложения
В качестве темы можно сделать web-приложение «Учёт заявок сотрудников». В нём пользователь через браузер сможет:

создавать заявку;

просматривать список заявок;

менять статус;

искать заявки по номеру и теме;

просматривать карточку заявки.

Структура БД в MS SQL Server может быть такой:

Users(id, login, password_hash, full_name, role)

Requests(id, request_date, author_id, subject, description, status, priority)

RequestHistory(id, request_id, change_date, old_status, new_status, comment)

Логика приложения:

страница входа;

главная страница со списком заявок;

форма добавления заявки;

страница редактирования статуса;

отчёт по заявкам за период.

Итоговая оценка
Web-архитектура для компании выгодна тем, что централизует данные, упрощает сопровождение и позволяет работать из браузера без установки толстого клиента. Для учебного или курсового проекта на Delphi 10.2 наиболее удачным вариантом будет простое ISAPI-приложение под IIS с MS SQL Server, потому что эта связка наглядно демонстрирует серверную web-архитектуру и соответствует требованиям задания
